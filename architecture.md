## 四、数据流图

### 4.1 音乐播放流程

```
1. 客户端请求音乐列表
   └── GET /api/v1/music

2. 后端查询 SQLite，返回音乐元数据
   └── { id, title, artist, duration, file_path }

3. 客户端选择播放
   └── GET /api/v1/music/{id}/stream

4. 后端读取本地文件，流式返回
   └── audio/mpeg streaming response

5. 客户端使用 ExoPlayer 播放
```

### 4.2 音乐上传流程（管理后台）

```
1. 管理员上传音乐文件
   └── POST /admin/music with multipart/form-data

2. 后端接收文件，提取元数据
   └── 使用 JAudioTagger 解析

3. 文件保存到本地 music-storage 目录

4. 元数据保存到 SQLite

5. 返回上传成功
```

### 4.3 歌单同步流程

```
1. 用户在客户端修改歌单
   └── POST /api/v1/playlists/{id}/songs

2. 后端更新数据库

3. 其他设备（若有）拉取最新数据
   └── GET /api/v1/playlists
```

## 六、技术栈总结

| 层级 | 技术 |
