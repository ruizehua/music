# 安全策略

> 本文件定义了音乐软件项目的安全策略，包括认证授权方案、数据加密、漏洞防护和密钥管理。

---

## 一、认证授权方案

### 1.1 用户认证

| 项目 | 说明 |
|------|------|
| 认证方式 | JWT Token |
| Token 类型 | Access Token + Refresh Token |
| Access Token 有效期 | 1 小时 |
| Refresh Token 有效期 | 7 天 |
| 密码加密 | BCrypt（强度 10） |

### 1.2 授权机制

- ✅ 基于角色的访问控制（RBAC）
- ✅ 每个 API 端点进行权限校验
- ✅ 用户只能访问自己的数据
- ✅ 资源级别的访问控制

---

## 二、数据加密

### 2.1 传输加密

| 协议 | 说明 |
|------|------|
| API 通信 | HTTPS |
| TLS 版本 | TLS 1.2+ |
| 证书类型 | 有效 SSL 证书 |

### 2.2 存储加密

| 数据类型 | 加密方式 |
|----------|----------|
| 用户密码 | BCrypt |
| 敏感配置 | 环境变量 |
| 日志 | 脱敏处理 |

---

## 三、漏洞防护

### 3.1 输入校验

- ✅ 参数校验（@Valid 注解）
- ✅ 文件类型校验
- ✅ 文件大小限制（50MB）
- ✅ SQL 注入防护（使用 ORM）

### 3.2 安全头

| 安全头 | 值 | 说明 |
|--------|-----|------|
| X-Content-Type-Options | nosniff | 防止 MIME 类型嗅探 |
| X-Frame-Options | DENY | 防止点击劫持 |
| X-XSS-Protection | 1; mode=block | 防止 XSS |
| Content-Security-Policy | default-src 'self' | 内容安全策略 |

---

## 四、密钥管理

### 4.1 密钥存储

| 密钥类型 | 存储方式 |
|----------|----------|
| JWT 密钥 | 环境变量 |
| API Key | 环境变量 |
| 数据库密码 | 环境变量 |

### 4.2 密钥轮换

| 密钥类型 | 轮换周期 |
|----------|----------|
| JWT 密钥 | 90 天 |
| API Key | 30 天 |

---

## 五、日志与审计

### 5.1 日志规范

- ✅ 禁止记录敏感信息（密码、Token）
- ✅ 记录关键操作日志
- ✅ 日志保留至少 7 天
- ✅ 日志级别：DEBUG、INFO、WARN、ERROR

### 5.2 审计日志

| 记录内容 | 说明 |
|----------|------|
| 操作时间 | ISO 8601 格式 |
| 操作人 | 用户 ID |
| 操作类型 | CRUD |
| 操作对象 | 资源 ID |
| 操作结果 | 成功/失败 |
