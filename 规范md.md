# Spec Coding 最大完备模板与 Git 协作规范

> 本文档定义了 Spec Coding 工作流中，用于企业级、长周期、高可靠性项目的**完整 Markdown 规范文件体系**，以及与之配套的 **Git 操作规范**。
> 
> 所有规范文档与源代码一同纳入 Git 版本控制，实现需求、设计、实现、验证的全链路可追溯。

---

## 一、项目规范文件清单（最大完备模板）

### 1.1 项目定义与需求层
| 文件名 | 作用 | 生成方式 |
| :--- | :--- | :--- |
| `README.md` | 项目总览、业务目标、快速上手指南、文档索引。 | 人类编写或 AI 辅助生成 |
| `glossary.md` | 术语表，统一领域语言，消除歧义。 | 人类编写或 AI 辅助生成 |
| `spec.md` | 详细需求规格说明书，包含用户故事、功能需求、业务规则、边界条件。 | AI 生成初稿，人类审查确认 |
| `non-functional.md` | 非功能需求：性能指标、安全要求、可用性、可扩展性、合规约束等。 | 人类主导编写 |
| `changelog.md` | **需求变更历史记录**，每次需求变更必须先更新此文件，记录变更时间、变更人、变更内容、变更原因。 | 人类编写 |

### 1.2 架构与设计层
| 文件名 | 作用 | 生成方式 |
| :--- | :--- | :--- |
| `design.md` 或 `plan.md` | 总体技术设计方案，涵盖架构决策、技术栈选型、模块划分等。 | AI 生成初稿，人类审查确认 |
| `architecture.md` | 系统架构详图（C4 模型、逻辑视图、物理视图）、模块/组件关系。 | AI 生成初稿，人类审查确认 |
| `api.md` | 接口规范，可用 Markdown 表格描述接口列表、请求/响应结构、错误码。 | AI 生成初稿，人类审查确认 |
| `data-model.md` 或 `schema.md` | 数据模型定义：实体、字段、关系、索引设计、数据库选型。 | AI 生成初稿，人类审查确认 |
| `ui-ux.md` | （如有前端）页面结构、交互流程、组件树、设计系统引用。 | 人类或设计师主导，AI 辅助 |

### 1.3 执行与实现层
| 文件名 | 作用 | 生成方式 |
| :--- | :--- | :--- |
| `tasks.md` | 按优先级和依赖关系拆解的原子化开发任务清单。 | AI 生成初稿，人类审查确认 |
| `code-style.md` | 代码风格指南、命名规范、ESLint/Prettier 配置说明、注释规范。 | 人类主导编写 |
| `contribution.md` | 开发约定：分支策略、提交信息规范、代码评审准则、PR 流程。 | 人类主导编写 |

### 1.4 验证与质量层
| 文件名 | 作用 | 生成方式 |
| :--- | :--- | :--- |
| `checklist.md` | 功能验收清单，定义每个需求的验收标准、测试场景和完成定义（DoD）。 | AI 生成初稿，人类审查确认 |
| `test-plan.md` | 测试策略与计划：测试金字塔、覆盖率目标、Mock 策略、测试环境。 | AI 生成初稿，人类审查确认 |
| `test-cases.md` | （可选拆分）关键集成测试用例或 E2E 场景的详细步骤与预期结果。 | AI 生成初稿，人类审查确认 |

### 1.5 运维与安全层
| 文件名 | 作用 | 生成方式 |
| :--- | :--- | :--- |
| `deployment.md` | 部署架构与环境说明：CI/CD 流程、环境变量、配置管理、日志监控。 | 人类主导编写 |
| `security.md` | 安全策略：认证授权方案、数据加密、漏洞防护、密钥管理、合规要求。 | 人类主导编写 |

### 1.6 治理与演进层
| 文件名 | 作用 | 生成方式 |
| :--- | :--- | :--- |
| `decisions.md` 或 `adr/` | 架构决策记录（ADR），记录关键决策的背景、选项、后果及状态。 | 人类主导编写 |
| `changelog.md` | 版本变更日志，记录每个版本的修改、修复和新增内容。 | AI 可根据提交历史自动生成 |
| `roadmap.md` | 产品路线图，未来版本规划及里程碑。 | 人类（产品/项目负责人）编写 |

## 二、AI 行为约定文件（元规则）

这些文件告诉 AI **如何工作**，而非具体做什么，是 Spec Coding 工作流的第一环。

| 文件名 | 作用 | 适用工具 |
| :--- | :--- | :--- |
| `AGENTS.md` | 跨工具通用项目指令，定义构建命令、测试命令、代码风格、不得修改的文件边界等。 | Codex CLI, Cursor, Copilot, Aider 等 |
| `CLAUDE.md` | Claude Code 专属项目备忘录，内容可与 `AGENTS.md` 相同（建议使用软链接）。 | Claude Code |
| `.cursorrules` | Cursor IDE 的项目级规则文件，内容可引用 `AGENTS.md`。 | Cursor |

**建议**：以 `AGENTS.md` 作为唯一事实来源，为不同工具创建软链接，如：
```bash
ln -s AGENTS.md CLAUDE.md
