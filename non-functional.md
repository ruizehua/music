# 非功能需求规范

> 本文件定义了音乐软件项目的非功能需求，包括性能指标、安全要求、可用性、可扩展性和合规约束等。

---

## 一、性能指标

### 1.1 响应时间
| 指标 | 要求 | 说明 |
|------|------|------|
| API 平均响应时间 | ≤ 200ms | 正常负载下的平均响应时间 |
| API P95 响应时间 | ≤ 500ms | 95% 请求的响应时间上限 |
| API P99 响应时间 | ≤ 1s | 99% 请求的响应时间上限 |
| 音乐列表加载时间 | ≤ 1s | 加载 100 首音乐列表 |
| 音乐播放启动时间 | ≤ 500ms | 从点击到开始播放 |

### 1.2 吞吐量
| 指标 | 要求 | 说明 |
|------|------|------|
| API 并发处理能力 | ≥ 100 req/s | 稳定运行的并发请求数 |
| 音乐文件上传速度 | ≥ 10MB/s | 单文件上传速率 |
| 流媒体传输速率 | ≥ 256kbps | 保证流畅播放的最低速率 |

### 1.3 资源使用
| 指标 | 要求 | 说明 |
|------|------|------|
| 服务端内存占用 | ≤ 512MB | 稳定运行时的内存使用 |
| 服务端 CPU 占用 | ≤ 50% | 正常负载下的 CPU 使用率 |
| 客户端内存占用 | ≤ 200MB | Android 应用运行时内存 |

---

## 二、安全要求

### 2.1 认证与授权
| 要求 | 说明 |
|------|------|
| 用户认证 | 使用 JWT Token 进行身份验证 |
| Token 有效期 | 访问 Token 有效期 1 小时，刷新 Token 有效期 7 天 |
| 密码策略 | 长度 ≥ 8 位，包含大小写字母和数字 |
| 密码存储 | 使用 BCrypt 算法加密存储 |

### 2.2 数据保护
| 要求 | 说明 |
|------|------|
| 传输加密 | 所有 API 通信使用 HTTPS |
| 敏感数据 | 用户密码、Token 等敏感信息禁止明文传输和存储 |
| 日志审计 | 记录关键操作日志，包含操作时间、操作人、操作内容 |

### 2.3 访问控制
| 要求 | 说明 |
|------|------|
| 权限校验 | 每个 API 端点进行权限校验 |
| 资源隔离 | 用户只能访问自己创建的播放列表 |
| 限流策略 | 单用户每分钟最多请求 60 次 API |

---

## 三、可用性

### 3.1 服务可用性
| 指标 | 要求 | 说明 |
|------|------|------|
| 服务端可用性 | ≥ 99.9% | 年度可用时间比例 |
| 故障恢复时间 | ≤ 5 分钟 | 服务故障后的恢复时间 |

### 3.2 客户端稳定性
| 要求 | 说明 |
|------|------|
| 崩溃率 | ≤ 0.1% | 应用崩溃次数占启动次数的比例 |
| ANR 率 | ≤ 0.05% | 应用无响应次数占启动次数的比例 |

### 3.3 兼容性
| 要求 | 说明 |
|------|------|
| Android 版本兼容 | API 26 (Android 8.0) 及以上 |
| 屏幕适配 | 支持主流屏幕分辨率（480x800 至 1440x3200） |
| 网络兼容 | 支持 WiFi、4G、5G 网络环境 |

---

## 四、可扩展性

### 4.1 架构扩展性
| 要求 | 说明 |
|------|------|
| 模块化设计 | 各模块职责单一，接口清晰 |
| 松耦合 | 模块间依赖最小化，便于独立升级 |
| 水平扩展 | 支持通过增加服务器节点扩展容量 |

### 4.2 数据扩展性
| 要求 | 说明 |
|------|------|
| 数据库扩展 | 支持数据库读写分离 |
| 文件存储扩展 | 支持扩展到分布式文件存储系统 |
| 缓存策略 | 使用缓存减少数据库压力 |

---

## 五、合规约束

### 5.1 隐私合规
| 要求 | 说明 |
|------|------|
| 用户同意 | 收集用户数据前需获得明确同意 |
| 数据删除 | 用户可随时删除自己的数据 |
| 隐私政策 | 提供清晰的隐私政策说明 |

### 5.2 版权合规
| 要求 | 说明 |
|------|------|
| 音乐来源 | 用户需保证上传的音乐有合法版权 |
| 使用范围 | 仅供个人使用，禁止商业传播 |

---

## 六、运维要求

### 6.1 监控与日志
| 要求 | 说明 |
|------|------|
| 健康检查 | 提供健康检查 API 端点 |
| 日志级别 | 支持 DEBUG、INFO、WARN、ERROR 级别 |
| 日志持久化 | 日志保留至少 7 天 |

### 6.2 配置管理
| 要求 | 说明 |
|------|------|
| 环境分离 | 开发、测试、生产环境分离 |
| 配置外部化 | 敏感配置通过环境变量或配置中心管理 |
