# 技术设计方案

> 本文件定义了音乐软件项目的总体技术设计方案，涵盖架构决策、技术栈选型、模块划分等。

---

## 一、架构决策

### 1.1 架构风格
- **架构模式**: 客户端-服务端分离架构（Client-Server）
- **设计原则**: 单一职责、开闭原则、依赖倒置
- **架构特点**: 松耦合、可扩展、易维护

### 1.2 关键决策

| 决策项 | 方案 | 理由 |
|--------|------|------|
| 后端框架 | Spring Boot 3.x | 社区成熟，生态完善，便于快速构建 RESTful 服务 |
| 数据库 | SQLite | 轻量级，无需独立服务，适合个人使用场景 |
| 前端框架 | Jetpack Compose | 现代 Android UI 框架，声明式编程，性能优秀 |
| 状态管理 | ViewModel + StateFlow | 符合 Android 架构组件最佳实践 |
| 网络请求 | Retrofit + OkHttp | 成熟稳定，功能强大 |
| 媒体播放 | ExoPlayer / Media3 | Google 官方媒体库，支持多种格式和流媒体 |

## 三、模块划分

### 3.1 后端模块

| 模块 | 职责 | 说明 |
|------|------|------|
| controller | REST API 控制层 | 处理 HTTP 请求，调用 Service |
| service | 业务逻辑层 | 实现核心业务逻辑 |
| repository | 数据访问层 | 数据库操作 |
| model | 数据模型 | 实体类定义 |
| dto | 数据传输对象 | API 请求/响应结构 |
| config | 配置类 | 应用配置 |
| exception | 异常处理 | 全局异常处理 |

### 3.2 客户端模块

| 模块 | 职责 | 说明 |
|------|------|------|
| ui | 用户界面层 | Compose 组件和屏幕 |
| viewmodel | 视图模型 | 管理 UI 状态 |
| data | 数据层 | 远程和本地数据源 |
| domain | 领域层 | 业务模型和用例 |
| di | 依赖注入 | Hilt 模块 |
| util | 工具类 | 通用工具函数 |

## 四、关键设计

### 4.1 数据流设计

**音乐上传流程**:
```
客户端上传 → Controller → Service → 文件存储 → 元数据提取 → 数据库保存 → 响应
```

**音乐播放流程**:
```
客户端请求 → Controller → Service → 数据库查询 → 文件读取 → 流式返回 → 客户端播放
```

### 4.2 安全设计

- **认证**: JWT Token 认证
- **授权**: 基于用户的资源访问控制
- **传输**: HTTPS 加密传输
- **存储**: 密码 BCrypt 加密存储
